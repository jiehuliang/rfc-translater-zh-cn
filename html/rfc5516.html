Abstract

摘要

This document registers a set of IANA Diameter Command Codes to be used in new vendor-specific Diameter applications defined for the Third Generation Partnership Project (3GPP) Evolved Packet System (EPS). These new Diameter applications are defined for Mobile Management Entity (MME)- and Serving GPRS (General Packet Radio Service) Support Node (SGSN)-related interfaces in in the architecture for the Evolved 3GPP Packet Switched Domain, which is also known as the Evolved Packet System (EPS).

本文档注册了一组IANA Diameter命令代码，用于为第三代合作伙伴关系项目（3GPP）演进包系统（EPS）定义的新供应商特定Diameter应用程序。这些新的Diameter应用被定义用于演进3GPP分组交换域（也称为演进分组系统（EPS））的架构中的移动管理实体（MME）和服务GPRS（通用分组无线电服务）支持节点（SGSN）相关接口。

1. Introduction
1. 介绍

The Third Generation Partnership Project (3GPP) is defining the Evolved 3GPP Packet Switched Domain - also known as the Evolved Packet System (EPS). As part of this architecture, the interfaces based on the Diameter protocol [RFC3588] require the definition of two new Diameter applications.

第三代合作伙伴计划（3GPP）正在定义演进的3GPP分组交换域-也称为演进的分组系统（EPS）。作为该体系结构的一部分，基于Diameter协议[RFC3588]的接口需要定义两个新的Diameter应用程序。

As defined in [TS29.272], the 3GPP S6a/S6d application (vendor-specific application id: 16777251) enables the transfer of subscriber-related data between the Mobile Management Entity (MME) and the Home Subscriber Server (HSS) on the S6a interface and between the Serving GPRS Support Node (SGSN) and the Home Subscriber Server (HSS) on the S6d interface.

如[TS29.272]中所定义，3GPP S6a/S6d应用程序（特定于供应商的应用程序id:16777251）能够在S6a接口上的移动管理实体（MME）和家庭用户服务器（HSS）之间以及服务GPRS支持节点（SGSN）和家庭用户服务器（HSS）之间传输与用户相关的数据在S6d接口上。

Also defined in [TS29.272], the 3GPP S13/S13' application (vendor-specific application id: 16777252) enables the Mobile Equipment Identity check procedure between the Mobile Management Entity (MME) and the Equipment Identity Register (EIR) on the S13 interface and between the Serving GPRS Support Node (SGSN) and the Equipment Identity Register (EIR) on the S13' interface.

也在[TS29.272]中定义，3GPP S13/S13'应用程序（特定于供应商的应用程序id:16777252）在移动管理实体（MME）和S13接口上的设备标识寄存器（EIR）之间以及服务GPRS支持节点（SGSN）和设备标识寄存器之间启用移动设备标识检查程序（EIR）在S13'接口上。

Both Diameter applications are defined under the 3GPP vendor-id "10415". This document defines the assigned values of the command codes used in these applications.

这两种直径应用都在3GPP供应商id“10415”下定义。本文档定义了这些应用程序中使用的命令代码的赋值。

2. Terminology
2. 术语

The base Diameter specification (Section 1.3 of [RFC3588]) defines most of the terminology used in this document. Additionally, the terms and acronyms defined in [TS29.272] are used in this document.

底座直径规范（RFC3588第1.3节）定义了本文件中使用的大部分术语。此外，本文件中使用了[TS29.272]中定义的术语和首字母缩略词。

3. Command Codes
3. 命令代码

The 3GPP S6a/S6d application described in Section 5 of [TS29.272] requires the allocation of command code values for the following command pairs:

[TS29.272]第5节中描述的3GPP S6a/S6d应用需要为以下命令对分配命令代码值：

o 3GPP-Update-Location-Request/Answer (ULR/ULA)

o 3GPP更新位置请求/应答（ULR/ULA）

o 3GPP-Cancel-Location-Request/Answer (CLR/CLA)

o 3GPP取消位置请求/应答（CLR/CLA）

o 3GPP-Authentication-Information-Request/ Answer (AIR/AIA)

o 3GPP认证信息请求/应答（AIR/AIA）

o 3GPP-Insert-Subscriber-Data-Request/Answer (IDR/IDA)

o 3GPP插入用户数据请求/应答（IDR/IDA）

o 3GPP-Delete-Subscriber-Data-Request/Answer (DSR/DSA)

o 3GPP删除用户数据请求/应答（DSR/DSA）

o 3GPP-Purge-UE-Request/Answer (PUR/PUA)

o 3GPP清除UE请求/应答（PUR/PUA）

o 3GPP-Reset-Request/Answer (RSR/RSA)

o 3GPP重置请求/应答（RSR/RSA）

o 3GPP-Notify-Request/Answer (NOR/NOA)

o 3GPP通知请求/应答（NOR/NOA）

The 3GPP S13/S13 application described in Section 6 of [TS29.272] requires the allocation of a command code value for the following command pair:

[TS29.272]第6节中描述的3GPP S13/S13应用要求为以下命令对分配命令代码值：

o 3GPP-ME-Identity-Check-Request/Answer (ECR/ECA)

o 3GPP ME身份检查请求/应答（ECR/ECA）

5. Security Considerations
5. 安全考虑

This document describes command codes used in applications that build on top of the Diameter base protocol and the same security considerations described in [RFC3588] are applicable to this document. No further extensions are required beyond the security mechanisms offered by [RFC3588].

本文档描述了构建在Diameter基础协议之上的应用程序中使用的命令代码，[RFC3588]中描述的相同安全注意事项适用于本文档。除[RFC3588]提供的安全机制外，无需进一步扩展。
