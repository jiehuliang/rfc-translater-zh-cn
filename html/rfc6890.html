Special-Purpose IP Address Registries

专用IP地址注册

Abstract

摘要

This memo reiterates the assignment of an IPv4 address block (192.0.0.0/24) to IANA. It also instructs IANA to restructure its IPv4 and IPv6 Special-Purpose Address Registries. Upon restructuring, the aforementioned registries will record all special-purpose address blocks, maintaining a common set of information regarding each address block.

本备忘录重申向IANA分配IPv4地址块（192.0.0.0/24）。它还指示IANA重组其IPv4和IPv6专用地址注册中心。重组后，上述登记处将记录所有专用地址块，维护关于每个地址块的一组通用信息。

This memo obsoletes RFCs 4773, 5156, 5735, and 5736.

本备忘录废除了RFC 4773、5156、5735和5736。

1. Introduction
1. 介绍

In order to support new protocols and practices, the IETF occasionally reserves an address block for a special purpose. For example, [RFC1122] reserves an IPv4 address block (0.0.0.0/8) to represent the local (i.e., "this") network. Likewise, [RFC4291] reserves an IPv6 address block (fe80::/10) to represent link-scoped unicast addresses.

为了支持新的协议和实践，IETF偶尔会为特殊目的保留一个地址块。例如，[RFC1122]保留IPv4地址块（0.0.0.0/8）以表示本地（即“此”）网络。同样，[RFC4291]保留一个IPv6地址块（fe80:：/10）以表示链路范围的单播地址。

Periodically, the IETF publishes an RFC that catalogs special-purpose address blocks. Currently, [RFC5735] catalogs all IPv4 special-purpose address blocks and [RFC5156] catalogs all IPv6 special-purpose address blocks.

IETF定期发布RFC，对专用地址块进行编目。目前，[RFC5735]编目所有IPv4专用地址块，[RFC5156]编目所有IPv6专用地址块。

[RFC5736] assigns an IPv4 address block (192.0.0.0/24) to IANA and instructs IANA to allocate special-purpose address blocks from this space. [RFC5736] also instructs IANA to create an IPv4 Special-Purpose Address Registry that records allocations from this address

[RFC5736]将IPv4地址块（192.0.0.0/24）分配给IANA，并指示IANA从此空间分配专用地址块。[RFC5736]还指示IANA创建一个IPv4专用地址注册表，用于记录来自此地址的分配

space. However, [RFC5736] does not instruct IANA to record special-purpose address block reservations from outside of the aforementioned space in the IPv4 Special-Purpose Address Registry.

空间但是，[RFC5736]没有指示IANA从IPv4专用地址注册表中上述空间之外记录专用地址块保留。

Likewise, [RFC2928] assigns an IPv6 address block (2001:0000::/23) to IANA and instructs IANA to allocate special-purpose address blocks from this space. [RFC4773] instructs IANA to create an IPv6 Special-Purpose Address Registry that records allocations from this address space. However, [RFC4773] does not instruct IANA to record special-purpose address block reservations from outside of the aforementioned space in the IPv6 Special-Purpose Address Registry.

同样，[RFC2928]将IPv6地址块（2001:0000:：/23）分配给IANA，并指示IANA从此空间分配专用地址块。[RFC4773]指示IANA创建一个IPv6专用地址注册表，该注册表记录来自此地址空间的分配。但是，[RFC4773]没有指示IANA从IPv6专用地址注册表中上述空间之外记录专用地址块保留。

This memo reiterates the assignment of an IPv4 address block (192.0.0.0/24) to IANA. It also instructs IANA to restructure its IPv4 and IPv6 Special-Purpose Address Registries. Specifically, this memo instructs IANA to record all special-purpose address blocks in the aforementioned registries. These include, but are not limited to, IPv4 allocations from 192.0.0.0/24 and IPv6 allocations from 2001:0000::/23. Furthermore, this memo defines:

本备忘录重申向IANA分配IPv4地址块（192.0.0.0/24）。它还指示IANA重组其IPv4和IPv6专用地址注册中心。具体而言，本备忘录指示IANA在上述注册表中记录所有专用地址块。这些包括但不限于192.0.0.0/24的IPv4分配和2001:0000:：/23的IPv6分配。此外，本备忘录规定：

o a common set of information that the registries will maintain regarding each special-purpose address block

o 注册中心将维护的关于每个专用地址块的公共信息集

o a common set of requirements for future entries

o 未来项目的一套通用要求

When the aforementioned registries include all special-purpose address blocks, [RFC5735] and [RFC5156] will become redundant with the registries. Therefore, this memo obsoletes [RFC5735] and [RFC5156]. Because this memo reiterates the assignment of 192.0.0.0/24 to IANA, and because it restructures the IPv4 Special-Purpose Address Registry, it obsoletes [RFC5736]. Finally, because this memo restructures the IPv6 Special-Purpose Address Registry, it obsoletes [RFC4773].

当上述注册表包含所有专用地址块时，[RFC5735]和[RFC5156]将与注册表冗余。因此，本备忘录废除了[RFC5735]和[RFC5156]。由于该备忘录重申了将192.0.0.0/24分配给IANA，并且由于它重组了IPv4专用地址注册表，因此它淘汰了[RFC5736]。最后，由于此备忘录重新构造了IPv6专用地址注册表，因此它淘汰了[RFC4773]。

2. IANA考虑
2.1. Assignment of an IPv4 Address Block to IANA
2.1. 将IPv4地址块分配给IANA

Table 7 of this document records the assignment of an IPv4 address block (192.0.0.0/24) to IANA for IETF protocol assignments. This address allocation to IANA is intended to support IETF protocol assignments. A more general view of the roles of IANA with respect to address allocation functions is documented in Sections 4.1 and 4.3 [RFC2860].

本文档的表7记录了将IPv4地址块（192.0.0.0/24）分配给IANA用于IETF协议分配的情况。IANA的地址分配旨在支持IETF协议分配。第4.1节和第4.3节[RFC2860]对IANA在地址分配功能方面的作用进行了更全面的介绍。

IANA has designated special-purpose address blocks in compliance with [RFC2860].

IANA已根据[RFC2860]指定了专用地址块。

2.2. Restructuring of the IPv4 and IPv6 Special-Purpose Address Registries

2.2. IPv4和IPv6专用地址注册表的重组

IANA has restructured the following registries:

IANA重组了以下登记处：

o IPv4 Special-Purpose Address Registry

o IPv4专用地址注册表

o IPv6 Special-Purpose Address Registry

o IPv6专用地址注册表

The IPv4 Special-Purpose Address Registry records all IPv4 special-purpose address blocks. These reservations include, but are not limited to, allocations from the 192.0.0.0/24 address block. Likewise, the IPv6 Special-Purpose Address Registry records all IPv6 special-purpose address blocks. These reservations include, but are not limited to, allocations from the 2001:0000::/23 address block.

IPv4专用地址注册表记录所有IPv4专用地址块。这些保留包括但不限于来自192.0.0.0/24地址块的分配。同样，IPv6专用地址注册表记录所有IPv6专用地址块。这些保留包括但不限于来自2001:0000:：/23地址块的分配。

Section 2.2.1 of this document describes information that both registries will maintain for each entry. Initially, IANA has populated the IPv4 Special-Purpose Address Registry with information taken from Section 2.2.2 of this document. Likewise, IANA has populated the IPv6 Special-Purpose Address Registry with information taken from Section 2.2.3 of this document.

本文件第2.2.1节描述了两个登记处将为每个条目保留的信息。最初，IANA使用本文档第2.2.2节中的信息填充IPv4专用地址注册表。同样，IANA已使用本文件第2.2.3节中的信息填充IPv6专用地址注册表。

IANA will update the aforementioned registries as requested in the "IANA Considerations" section of a document that has passed IETF Review [RFC5226]. The "IANA Considerations" section must include all of the information specified in Section 2.2.1 of this document.

IANA将按照通过IETF审查的文件[RFC5226]中“IANA注意事项”部分的要求更新上述登记册。“IANA注意事项”部分必须包括本文件第2.2.1节中规定的所有信息。

2.2.1. Information Requirements
2.2.1. 信息要求

The IPv4 and IPv6 Special-Purpose Address Registries maintain the following information regarding each entry:

IPv4和IPv6专用地址注册中心维护有关每个条目的以下信息：

o Address Block - A block of IPv4 or IPv6 addresses that has been registered for a special purpose.

o 地址块-已为特殊目的注册的IPv4或IPv6地址块。

o Name - A descriptive name for the special-purpose address block.

o 名称-专用地址块的描述性名称。

o RFC - The RFC through which the special-purpose address block was requested.

o RFC—请求专用地址块的RFC。

o Allocation Date - The date upon which the special-purpose address block was allocated.

o 分配日期-分配专用地址块的日期。

o Termination Date - The date upon which the allocation is to be terminated. This field is applicable for limited-use allocations only.

o 终止日期-分配终止的日期。此字段仅适用于有限使用分配。

o Source - A boolean value indicating whether an address from the allocated special-purpose address block is valid when used as the source address of an IP datagram that transits two devices.

o Source—一个布尔值，指示当用作传输两个设备的IP数据报的源地址时，分配的专用地址块中的地址是否有效。

o Destination - A boolean value indicating whether an address from the allocated special-purpose address block is valid when used as the destination address of an IP datagram that transits two devices.

o Destination（目标）—一个布尔值，指示当用作传输两个设备的IP数据报的目标地址时，分配的专用地址块中的地址是否有效。

o Forwardable - A boolean value indicating whether a router may forward an IP datagram whose destination address is drawn from the allocated special-purpose address block between external interfaces.

o Forwardable（可转发）—一个布尔值，指示路由器是否可以转发IP数据报，该IP数据报的目标地址来自外部接口之间分配的专用地址块。

o Global - A boolean value indicating whether an IP datagram whose destination address is drawn from the allocated special-purpose address block is forwardable beyond a specified administrative domain.

o 全局-一个布尔值，指示目标地址来自分配的专用地址块的IP数据报是否可转发到指定的管理域之外。

o Reserved-by-Protocol - A boolean value indicating whether the special-purpose address block is reserved by IP, itself. This value is "TRUE" if the RFC that created the special-purpose address block requires all compliant IP implementations to behave in a special way when processing packets either to or from addresses contained by the address block.

o 由协议保留-一个布尔值，指示专用地址块是否由IP本身保留。如果创建专用地址块的RFC要求所有兼容的IP实现在处理到地址块包含的地址或从地址块包含的地址发送的数据包时以特殊方式运行，则此值为“TRUE”。

If the value of "Destination" is FALSE, the values of "Forwardable" and "Global" must also be false.

如果“Destination”的值为FALSE，“Forwardable”和“Global”的值也必须为FALSE。

3. Security Considerations
3. 安全考虑

Security of the Internet's routing system relies on the ability to authenticate an assertion of unique control of an address block. Measures to authenticate such assertions rely on validation that the address block forms part of an existing allocated address block and that there is a trustable and unique reference in the IANA address registries.

Internet路由系统的安全性依赖于对地址块的唯一控制断言进行身份验证的能力。验证此类断言的措施依赖于验证地址块是否构成现有已分配地址块的一部分，以及IANA地址注册表中是否存在可信任且唯一的引用。

The proposed registry is intended to provide an authoritative source of information regarding the currency and intended purpose of special purpose address blocks that are designated from the IANA-administered Special-Purpose registry. This is a small step towards the creation of a comprehensive registry framework that can be used as a trust point for commencing a chain of address validation. Consideration should be given to IANA registry publication formats that are machine parsable. Additionally, consideration should be given to the use of file signatures and associated certificate mechanisms to allow applications to confirm that the registry contents are current and that they have been published by the IANA.

拟议的注册旨在提供有关IANA管理的特殊用途注册指定的特殊用途地址块的货币和预期用途的权威信息来源。这是创建一个全面的注册表框架的一小步，该框架可用作启动地址链验证的信任点。应考虑IANA注册表可由机器解析的发布格式。此外，应考虑使用文件签名和相关证书机制，以允许应用程序确认注册表内容是最新的，并且已由IANA发布。
