The IPv6-Specific MIB Modules Are Obsolete

Abstract

摘要

In 2005-2006, the IPv6 MIB update group published updated versions of the IP-MIB, UDP-MIB, TCP-MIB, and IP-FORWARD-MIB modules, which use the InetAddressType/InetAddress construct to handle IPv4 and IPv6 in the same table. This document contains versions of the obsoleted IPV6-MIB, IPV6-TC, IPV6-ICMP-MIB, IPV6-TCP-MIB, and IPV6-UDP-MIB modules for the purpose of updating MIB module repositories. This document obsoletes RFCs 2452, 2454, 2465, and 2466 (i.e., the RFCs containing these MIBs) and reclassifies them as Historic.

在2005-2006年间，IPv6 MIB更新组发布了IP-MIB、UDP-MIB、TCP-MIB和IP-FORWARD-MIB模块的更新版本，这些模块使用InetAddressType/InetAddress构造在同一个表中处理IPv4和IPv6。本文档包含已淘汰的IPV6-MIB、IPV6-TC、IPV6-ICMP-MIB、IPV6-TCP-MIB和IPV6-UDP-MIB模块的版本，用于更新MIB模块存储库。本文件淘汰了RFC 2452、2454、2465和2466（即包含这些MIB的RFC），并将其重新分类为历史记录。

1. Motivation
1. 动机

In 2005-2006, the IPv6 MIB update group published updated versions of the IP-MIB [RFC4293], UDP-MIB [RFC4113], TCP-MIB [RFC4022], and IP-FORWARD-MIB [RFC4292] modules, which use the InetAddressType/ InetAddress construct to handle IPv4 and IPv6 in the same table. The RFC Editor marked these documents as obsoleting the corresponding IPV6-MIBs, but the extracted content of these MIBs never changed in MIB repositories, and the original RFCs (as is normal IETF policy) never changed from being Proposed Standard.

在2005-2006年间，IPv6 MIB更新组发布了IP-MIB[RFC4293]、UDP-MIB[RFC4113]、TCP-MIB[RFC4022]和IP-FORWARD-MIB[RFC4292]模块的更新版本，这些模块使用InetAddressType/InetAddress构造在同一个表中处理IPv4和IPv6。RFC编辑器将这些文档标记为淘汰相应的IPV6 MIB，但这些MIB的提取内容在MIB存储库中从未更改，并且原始RFC（通常为IETF策略）从未更改为建议的标准。

Note that the timeline of these MIB modules is as shown below (and it is the added support for IPv6 in the later revision of the original modules that people often overlook).

请注意，这些MIB模块的时间线如下所示（人们经常忽略的是在原始模块的后续版本中添加的对IPv6的支持）。 

                          IPv6-MIB--------X
                                           \
                 IP-MIB-------------------IP-MIB--->
        
                          IPv6-MIB--------X
                                           \
                 IP-MIB-------------------IP-MIB--->

This causes an unclear situation when simply looking at MIB repositories, so we are simply republishing these MIB modules with the Structure of Management Information (SMI) status changed to obsolete. This is an unusual step, and it is not the intended path with every obsolete MIB module; the special history of these modules led to this special step.

这导致在简单查看MIB存储库时出现不清楚的情况，因此我们只是在管理信息（SMI）状态的结构更改为过时的情况下重新发布这些MIB模块。这是一个不寻常的步骤，它不是每个过时的MIB模块的预期路径；这些模块的特殊历史导致了这一特殊步骤。
