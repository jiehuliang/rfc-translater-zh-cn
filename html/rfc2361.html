Abstract

摘要

Internet applications may reference specific codecs within the WAVE and AVI registries as follows: * video/vnd.avi; codec=XXX identifies a specific video codec (i.e., XXX) within the AVI Registry. * audio/vnd.wave; codec=YYY identifies a specific audio codec (i.e., YYY) within the WAVE Registry.

互联网应用程序可参考WAVE和AVI注册中的特定编解码器，如下所示：*video/vnd.AVI；codec=XXX标识AVI注册表中的特定视频编解码器（即XXX）。*音频/vnd.wave；codec=YYY标识WAVE注册表中的特定音频编解码器（即YYY）。

Appendix A and Appendix B provides an authoritative reference for the interpretation of the required "codec" parameter. That is, the current set of audio codecs that are registered within the WAVE Registry are enumerated in Appendix A. Appendix B enumerates the current set of video codecs that have been registered to date within the AVI Registry.

附录A和附录B提供了解释所需“编解码器”参数的权威参考。也就是说，在WAVE注册表中注册的当前音频编解码器集在附录A中列出。附录B列出了迄今为止在AVI注册表中注册的当前视频编解码器集。

1 Introduction

1导言

Internet-oriented multimedia applications reference multimedia content via predefined mechanisms (e.g., [2]). In the general case, this content was created primarily for the use of these Internet-oriented applications. Unfortunately, this Internet-oriented multimedia content represents a small minority of the total amount of multimedia content that has been created to date.

面向Internet的多媒体应用程序通过预定义的机制（例如[2]）引用多媒体内容。在一般情况下，创建此内容主要是为了使用这些面向Internet的应用程序。不幸的是，这种面向互联网的多媒体内容只占迄今为止创建的多媒体内容总量的一小部分。

For this reason, a growing interest is forming in establishing mechanisms by which the repertoire of multimedia content available to Internet-oriented applications(e.g., for RTSP [3]) may be greatly extended to include multimedia content that has been created outside of distinctly Internet contexts. For this to occur, a mechanism must

因此，人们对建立机制越来越感兴趣，通过这些机制，面向互联网的应用程序（例如，RTSP[3]）可以使用的多媒体内容库可以大大扩展，以包括在互联网环境之外创建的多媒体内容。要实现这一点，必须有一个机制

be created for Internet protocols (e.g., [1], [3], [4]) to be able to identify the codecs by which this so-called "traditional" multimedia content has been encoded.

为互联网协议（例如，[1]、[3]、[4]）创建，以便能够识别编码此所谓“传统”多媒体内容的编解码器。

Unfortunately, several distinct encoding systems exist for traditional multimedia content. Each system has its own registry to ensure unique and stable codec identifications within that system. Perhaps the best known of these registries are Microsoft (for WAVE and AVI content) and Apple (for QuickTime content).

不幸的是，传统多媒体内容存在几种不同的编码系统。每个系统都有自己的注册表，以确保该系统中唯一和稳定的编解码器标识。也许这些注册中心中最著名的是微软（WAVE和AVI内容）和苹果（QuickTime内容）。

The purpose of this paper is to establish a mechanism by which codecs registered within Microsoft's WAVE and AVI Registries may be referenced within the IANA Namespace by Internet applications.

本文的目的是建立一种机制，通过该机制，互联网应用程序可以在IANA名称空间中引用在Microsoft WAVE和AVI注册中心注册的编解码器。

2 References to Registries within the IANA Vendor Tree

2对IANA供应商树中的注册中心的引用

Reference [7] specifies that the IANA Namespace encompasses several trees. Discussions within the IETF-Types mailing list concluded that the most appropriate tree in which to reference codecs, which had already been registered by non-IANA Registries, is the Vendor Tree.

参考文献[7]指定IANA命名空间包含多个树。IETF类型邮件列表中的讨论得出结论，参考编解码器的最合适的树（已由非IANA注册中心注册）是供应商树。

As a result, the non-IANA registry is identified within the IANA Vendor tree by vnd.RegistryName. A specific codec, which has been registered within that registry, is identified by a required codec parameter as specified by Section 2.2.3 of [7].

因此，非IANA注册表在IANA供应商树中由vnd.RegistryName标识。已在该注册表中注册的特定编解码器由[7]第2.2.3节规定的所需编解码器参数标识。

3 WAVE and AVI Registries

3 WAVE和AVI注册中心

Both the WAVE and AVI Registries are historic databases that have been maintained by Microsoft as a free service. The Registries sought to assist developers of WAVE and AVI content and to standardize WAVE and AVI content by 1) avoiding conflict and/or duplication with current definitions, and 2) providing the registered information in a standard document and format that is publicly available. The historic nature of these databases implies that unless the original registrants informed the registrar of a change of status (e.g., company acquired, new contact, new location, new phone), the contact information has generally not been updated from the originally registered values.

WAVE和AVI注册表都是历史数据库，由Microsoft作为免费服务维护。注册中心试图帮助WAVE和AVI内容的开发人员，并通过1）避免与当前定义的冲突和/或重复，以及2）以公开的标准文档和格式提供注册信息来标准化WAVE和AVI内容。这些数据库的历史性质意味着，除非原始注册人通知注册人状态发生变化（例如，公司收购、新联系人、新地点、新电话），否则通常不会根据原始注册值更新联系信息。

Audio codecs within the WAVE Registry are identified by WAVE Format IDs. The (audio) WAVE Format ID is officially known as "WAVE form Registration Number". The WAVE Format ID is a hexadecimal integer value. These codecs may be referenced within the IANA namespace as

WAVE注册表中的音频编解码器由WAVE格式ID标识。（音频）波形格式ID正式称为“波形注册号”。波形格式ID是十六进制整数值。这些编解码器可以在IANA名称空间中作为参考

"audio/vnd.wave; codec=XXX", where XXX represents a valid WAVE Format ID (e.g., the WAVE Format ID of "123" is referenced within the IANA namespace by "audio/vnd.wave; codec=123").

“audio/vnd.wave；codec=XXX”，其中XXX表示有效的波形格式ID（例如，“123”的波形格式ID在IANA名称空间中由“audio/vnd.wave；codec=123”引用）。

Video codecs within the AVI Registry are identified by AVI Codec IDs. The AVI Codec ID value is a FourCC encoding. A FourCC is 32-bits long and represents a (case-sensitive) four-character (i.e., ASCII) code value. These codecs may be referenced within the IANA namespace as "video/vnd.avi; codec=XXX", where XXX represents a valid AVI Codec ID (e.g., the WAVE Format ID of "SCRN" is referenced within the IANA namespace by "video/vnd.avi; codec=SCRN").

AVI注册表中的视频编解码器由AVI编解码器ID标识。AVI编解码器ID值是FourCC编码。FourCC的长度为32位，表示（区分大小写）四个字符（即ASCII）的代码值。这些编解码器可以在IANA名称空间中被引用为“video/vnd.avi；codec=XXX”，其中XXX表示有效的avi编解码器ID（例如，“SCRN”的波形格式ID在IANA名称空间中被“video/vnd.avi；codec=SCRN”引用）。

附录A是已在WAVE注册表中注册（截至1998年1月）的整套音频编解码器的权威列表。附录B是在AVI注册中心注册的（截至1998年1月）整套视频编解码器的权威列表。

4 Mapping Codec IDs to GUID Values

4将编解码器ID映射到GUID值

Direct mappings exist between WAVE Format IDs and GUIDs and between FourCC codec values and GUIDs [5]. [Note: GUIDs are Globally Unique Identifiers that are also known as Universally Unique Identifiers (UUIDs). UUIDs have been standardized within the Open Software Foundation's (OSF) Distributed Computing Environment (DCE).] These mappings enable GUID-oriented software to directly refer to these historic codec values. For example, the Advanced Streaming Format (ASF) [6] uses GUID values to refer to codecs, and the following mechanism is used to convert the historic WAVE and AVI codec values into the appropriate GUID value for use within ASF.

WAVE格式ID和GUID之间以及FourCC编解码器值和GUID之间存在直接映射[5]。[注意：GUID是全局唯一标识符，也称为通用唯一标识符（UUID）。UUID已在开放软件基金会（OSF）分布式计算环境（DCE）中标准化。]这些映射使面向GUID的软件能够直接引用这些历史编解码器值。例如，高级流格式（ASF）[6]使用GUID值引用编解码器，以下机制用于将历史WAVE和AVI编解码器值转换为适当的GUID值，以便在ASF中使用。

WAVE Format IDs are converted to GUIDs by inserting the hexadecimal value of the WAVE Format ID into the XXXXXXXX part of the following template: {XXXXXXXX-0000-0010-8000-00AA00389B71}. For example, a WAVE Format ID of 123 has the GUID value of {00000123-0000-0010-8000- 00AA00389B71}.

通过将波形格式ID的十六进制值插入以下模板的XXXXXXXX部分，{XXXXXXXX-0000-0010-8000-00AA00389B71}，可以将波形格式ID转换为GUID。例如，波格式ID为123的GUID值为{00000123-0000-0010-8000-00AA00389B71}。

FourCC values are converted to GUIDs by inserting the FourCC value into the XXXXXXXX part of the same template: {XXXXXXXX-0000-0010- 8000-00AA00389B71}. For example, a conversion of the FourCC value of "H260" would result in the GUID value of {30363248-0000-0010-8000- 00AA00389B71}. [Note: the 32-bit FourCC value of "H260" is converted into hexadecimal 32-bit value (i.e., 30363248) because the initial XXXXXXXX of the GUID is defined as a DWORD and thus takes a 32-bit hexadecimal value. Endian considerations account for the apparent re-ordering of the original ASCII text.]

通过将FourCC值插入同一模板的XXXXXXXX部分：{XXXXXXXX-0000-0010-8000-00AA00389B71}，FourCC值被转换为GUID。例如，FourCC值“H260”的转换将导致GUID值{30363248-0000-0010-8000-00AA00389B71}。[注：“H260”的32位FourCC值转换为十六进制32位值（即30363248），因为GUID的初始XXXXXXXX定义为DWORD，因此采用32位十六进制值。Endian考虑了原始ASCII文本的明显重新排序。]
